BGP邻居


原理概述

路由协议通常分为内部网关协议 IGP （interior Gateway Protocol）和外部网关协议EGP （exterior Gateway Protocol）两大类。一般来讲 IGP用于AS内部，而EGP用于AS之间。常见的IGP包含RIP、OSPF、IS-IS、常见的EGP只有BGP了。

BGP有许多版本，现在使用的为BGP-4 （RFC4271） ，在此之前BGP还有3个版本均已停止使用。

BGP基于TCP建立连接 端口号为179，可以跨越设备建立邻居关系。BGP是一种动态路由协议，其主要功能是完成最佳的路由选择并在BGP邻居之间进行最佳路由的传递。

BGP支持无类域间路由CIDR （Classless Inter-Domain Routing）,采用增量更新方式，不进行周期性更新，大大地减少了BGP在传递路由时所占用的带宽，适用于互联网传递大量的路由信息。

BGP提供了丰富的路由属性 （Attribute），通过对这些属性的操作和控制，BGP能够非常容易地实现丰富的路由策略，BGP的邻居关系分为IBGP （Internal BGP）和EBGP （External BGP），在建立邻居时，AS号一致，为IBGP，不一致为EBGP。BGP没有自动建立邻居的能力，邻居关系需要手工配置来建立。

实验内容

R1与R2属于同一个运行商网络，AS编号为100，R1与R2之间的邻居关系为 IBGP邻居关系。R3属于另一个运营商网络，AS编号为200，R3与R2之间为EBGP邻居关系，路由器采用物理接口与Loopback接口来建立IBGP和EBGP邻居关系。

实验拓扑

实验步骤

1.基本配置

配置基本接口地址，并使用ping命令检测R1与R2 (其余直连网段的连通性测试在此省略)

[R1]ping 10.0.12.2
PING 10.0.12.2: 56  data bytes, press CTRL_C to break
    Reply from 10.0.12.2: bytes=56 Sequence=1 ttl=255 time=50 ms
    Reply from 10.0.12.2: bytes=56 Sequence=2 ttl=255 time=50 ms
    Reply from 10.0.12.2: bytes=56 Sequence=3 ttl=255 time=10 ms
    Reply from 10.0.12.2: bytes=56 Sequence=4 ttl=255 time=40 ms
    Reply from 10.0.12.2: bytes=56 Sequence=5 ttl=255 time=30 ms

--- 10.0.12.2 ping statistics ---
    5 packet(s) transmitted
    5 packet(s) received
    0.00% packet loss
    round-trip min/avg/max = 10/36/50 ms
2.IBGP配置

在R1和R2上使用直连物理接口来建立IBGP邻居关系。为了实现链路冗余，R1与R2之间部署了两条链路。当一跳链路出现故障时，另一条物理链路可以提供连通性。

[R1]bgp 100
    [R1-bgp]peer 10.0.12.2 as-number 100
    [R1-bgp]peer 10.0.21.2 as-number 100

    [R2]bgp 100
    [R2-bgp]peer 10.0.12.1 as-number 100 
    [R2-bgp]peer 10.0.21.1 as-number 100

配置完IBGP邻居后 我们使用display bgp peer 命令 ，查看BGP的邻居关系。

[R2]dis bg peer 

    BGP local router ID : 10.0.12.2
    Local AS number : 100
    Total number of peers : 2                Peers in established state : 2

    Peer            V          AS  MsgRcvd  MsgSent  OutQ  Up/Down       State PrefRcv

    10.0.12.1       4         100       39       40     0 00:36:26 Established       1
    10.0.21.1       4         100       39       40     0 00:36:17 Established       1

通过display bgp peer 命令 我们可以看到R2 与 R1 的State为Established状态，表示邻居关系已经建立成功

接下来我们将loopback接口的地址宣告进IBGP使其完成通信

[R1-bgp]net 10.0.1.1 32
    [R2-bgp]net 10.0.2.2 32

我们使用display bgp routing-table来查看是否获取到对方的loopback接口的地址

[R1]display bgp routing-table 

    BGP Local router ID is 10.0.1.1 
    Status codes: * - valid, > - best, d - damped,
                h - history,  i - internal, s - suppressed, S - Stale
                Origin : i - IGP, e - EGP, ? - incomplete


    Total Number of Routes: 3
        Network            NextHop        MED        LocPrf    PrefVal Path/Ogn

    *>   10.0.1.1/32        0.0.0.0         0                     0      i
    *>i  10.0.2.2/32        10.0.12.2       0          100        0      i
    * i                     10.0.21.2       0          100        0      i

在此R1拥有一条去往R2 loopback接口的地址：10.0.2.2。而这条路由的下一跳 （nexthop） 有两条，10.0.21.2做冗余备份，所以不是最优的，同样也不会放进ip routing-table里。


3.EBGP配置

从前面的实验内容我们知道，使用物理接口来建立R1和R2的BGP邻居关系时，配置相对简单，并且能够实现邻居关系的冗余。但是，这样的配置会产生两个TCP会话和两个BGP邻居关系，当有路由需要彼此通告时，会通过这两个邻居关系分别进行通告，因此比较消耗设备资源，并且链路的不稳定也会导致BGP邻居关系的不稳定。

BGP是基于TCP建立连接的，如果我们需要通过loopback0建立TCP会话 首先得确保路由的可达性，所以我们首先配置静态路由，使其完成loopback之间的通信

[R2]ip route-static 10.0.3.3 32 10.0.23.3
    [R2]ip route-static 10.0.3.3 32 10.0.32.3

    [R3]ip route-static 10.0.2.2 32 10.0.23.2 
    [R3]ip route-static 10.0.2.2 32 10.0.32.2

测试连通性

[R3]ping -a 10.0.3.3 10.0.2.2 

    PING 10.0.2.2: 56  data bytes, press CTRL_C to break
        Reply from 10.0.2.2: bytes=56 Sequence=1 ttl=255 time=60 ms
        Reply from 10.0.2.2: bytes=56 Sequence=2 ttl=255 time=30 ms
        Reply from 10.0.2.2: bytes=56 Sequence=3 ttl=255 time=30 ms
        Reply from 10.0.2.2: bytes=56 Sequence=4 ttl=255 time=10 ms

    --- 10.0.2.2 ping statistics ---
        4 packet(s) transmitted
        4 packet(s) received
        0.00% packet loss
        round-trip min/avg/max = 10/32/60 ms

接下来我们将在R2和R3上使用Loopback 0接口来建立EBGP邻居关系。

[R2]bgp 100
    [R2-bgp]peer 10.0.3.3 as-number 200   
    [R2-bgp]peer 10.0.3.3 connect-interface lo0
    [R2-bgp]peer 10.0.3.3 ebgp-max-hop 2

    [R3]bgp 200
    [R3-bgp]peer 10.0.2.2 as-number 100   
    [R3-bgp]peer 10.0.2.2 connect-interface lo0
    [R3-bgp]peer 10.0.2.2 ebgp-max-hop 2

默认情况下，BGP会使用去往邻居路由器的出接口IP地址作为BGP报文的更新地址，R2的出口地址为10.0.23.2和10.0.32.2，与R3指定的邻居地址不一致，从而导致了无法建立邻居。所以我们使用connect-interface将更新源指定为lo0 不然TCP建立不成功

默认情况下EBGP邻居之间在发送BGP报文时，TTL值为1，EBGP默认情况下必须物理直连，但是R2和R3采用loopback 0接口建立邻居关系，TTL值会多减一次，成为0这个BGP报文就会被丢弃，所以我们需要使用EBGP-MAX-HOP 2 设置我们的TTL值为2，就可以正常建立邻居

配置完EBGP邻居后 我们使用display bgp peer 命令 ，查看EBGP的邻居关系。

[R2]dis bg peer 

    BGP local router ID : 10.0.12.2
    Local AS number : 100
    Total number of peers : 3                Peers in established state : 3

    Peer            V          AS  MsgRcvd  MsgSent  OutQ  Up/Down       State PrefRcv

    10.0.3.3        4         200       18       20     0 00:16:26 Established       0
    10.0.12.1       4         100       89       90     0 01:26:41 Established       1
    10.0.21.1       4         100       89       90     0 01:26:32 Established       1

这里能够看到 R2 与 R3 的state为Established，这样R2与R3的EGBP邻居就建立完成了。
