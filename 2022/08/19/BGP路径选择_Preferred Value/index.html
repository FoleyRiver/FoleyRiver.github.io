BGP路径选择——Preferred Value

原理概述

当一台BGP路由器中存在多条去往同一条目的地网络的BGP路由时，BGP协议会对这些BGP路由的属性进行比较，以确定去往该网络的最优BGP路由，然后将该最优BGP路由与其他路由协议进行比较，从而决定是否将该条最优BGP路由放进IP路由表中。路由器最终是根据IP路由表进行实际的报文转发的。在对BGP路由属性进行比较时，BGP会遵循一定的先后顺序进行比较，直到确定一条最优路由为止。在BGP路由属性的比较过程中，首先要比较的就是路由信息首选值Preferred Value，也简称PreVal。

路由信息的首选值Preferred Value的取值范围是0~65535,取值越大，优先级越高。缺省情况下，Preferred Value取值为0；通过修改Preferred Value的值，可以很方便地实现对路径选择的控制。Preferred Value 属性不会发送给任何BGP邻居，仅作为本地路由器用来选择最佳BGP路径之用。

实验内容

R1、R2、R3、R4全部运行BGP协议，使用Loopback 0接口建立邻居关系 AS 100使用OSPF搭建IGP，R1、R2与R4间使用静态建立Loopback可达性。R3宣告Lo1接口，R4宣告Lo1、Lo2接口。通过Preferred Value，使其R3去往Lo1接口去往R4的Lo1、Lo2分别通过R1与R2进行转发，从而实现流量分担，并互为备份。

实验拓扑

实验步骤

1.基础配置

首先我们在R1、R2、R3中配置OSPF，确保彼此Lo0口的可达性

[R1] ospf 1 router-id 1.1.1.1
    [R1-ospf-1] area 0.0.0.0
    [R1-ospf-1-area-0.0.0.0] network 10.0.13.0 0.0.0.255
    [R1-ospf-1-area-0.0.0.0] network 10.0.1.0 0.0.0.255

    [R2] ospf 1 router-id 2.2.2.2
    [R2-ospf-1] area 0.0.0.0
    [R2-ospf-1-area-0.0.0.0] network 10.0.23.0 0.0.0.255
    [R2-ospf-1-area-0.0.0.0] network 10.0.2.0 0.0.0.255

    [R3] ospf 1 router-id 3.3.3.3
    [R3-ospf-1] area 0.0.0.0
    [R3-ospf-1-area-0.0.0.0] network 10.0.13.0 0.0.0.255
    [R3-ospf-1-area-0.0.0.0] network 10.0.3.0 0.0.0.255
    [R3-ospf-1-area-0.0.0.0] network 10.0.23.0 0.0.0.255

2.BGP配置

我们对R1、R2、R3之间建立IBGP邻居关系、R2-R4、R1-R4建立EBGP邻居关系。

[R1] bgp 100
    [R1-bgp] peer 10.0.2.2 as-number 100
    [R1-bgp] peer 10.0.2.2 connect-interface lo0
    [R1-bgp] peer 10.0.2.2 next-hop-local
    [R1-bgp] peer 10.0.4.4 as-number 200
    [R1-bgp] peer 10.0.4.4 connect-interface lo0
    [R1-bgp] peer 10.0.4.4 ebgp-max-hop 2
    [R1-bgp] peer 10.0.3.3 as-number 100
    [R1-bgp] peer 10.0.3.3 connect-interface lo0
    [R1-bgp] peer 10.0.3.3 next-hop-local

    [R2] bgp 100
    [R1-bgp] peer 10.0.1.1 as-number 100
    [R1-bgp] peer 10.0.1.1 connect-interface lo0
    [R1-bgp] peer 10.0.1.1 next-hop-local
    [R2-bgp] peer 10.0.4.4 as-number 200
    [R2-bgp] peer 10.0.4.4 connect-interface lo0
    [R2-bgp] peer 10.0.4.4 ebgp-max-hop 2
    [R2-bgp] peer 10.0.3.3 as-number 100
    [R2-bgp] peer 10.0.3.3 connect-interface lo0
    [R2-bgp] peer 10.0.3.3 next-hop-local

    [R3] bgp 100
    [R3-bgp] peer 10.0.1.1 as-number 100
    [R3-bgp] peer 10.0.1.1 connect-interface LoopBack0
    [R3-bgp] peer 10.0.2.2 as-number 100
    [R3-bgp] peer 10.0.2.2 connect-interface LoopBack0
    [R3-bgp] network 10.0.33.33 255.255.255.255

    [R4] bgp 200
    [R4-bgp] peer 10.0.1.1 as-number 100
    [R4-bgp] peer 10.0.1.1 ebgp-max-hop 2
    [R4-bgp] peer 10.0.1.1 connect-interface LoopBack0
    [R4-bgp] peer 10.0.2.2 as-number 100
    [R4-bgp] peer 10.0.2.2 ebgp-max-hop 2
    [R4-bgp] peer 10.0.2.2 connect-interface LoopBack0
    [R4-bgp] network 10.0.44.44 255.255.255.255
    [R4-bgp] network 10.0.55.55 255.255.255.255

    *>i  10.0.55.55/32      10.0.1.1        0          100        0      200i
    * i                     10.0.2.2        0          100        0      200i

在此我们可以看到，在R3的BGP路由表中，去往10.0.44.44/32和10.0.55.55/32网络的路由条目各有两条，但是最终R3优选的都是下一跳为10.0.1.1的路由。这两条BGP路由信息的下一跳不同，但路由信息首选值PreVal、本地优先级LocPrf、路由生产方式、AS-path属性、Origin属性、MED属性、BGP对等体类型（IBGP邻居或者EBGP邻居）等都是相同的，所以最终BGP选择了router-id较小的R1 发布的路由作为最佳的路由。

在R1 R2的BGP路由表中，去往0.0.44.44/32和10.0.55.55/32网络的路由条目同样各有两条，但是选择的都是10.0.4.4的路由，这两条BGP路由信息的下一跳不同，他们的其他属性都是一样的，除了BGP对等体不同，所以最终R1 和R2都选择了EBGP邻居R4那里接收到的路由作为最佳路由。

无论是通过EBGP还是IBGP学习到的BGP路由条目，在BGP路由表中PrefVal都显示为缺省值0.

3.修改Preferred Value

目前，R3去往网络10.0.44.44/32和10.0.55.55/32的最佳下一跳都为R1，R2没有分担任何流量。接下来我们在R3上进行Preferred Value值的修改，使得R2分担从R3去往10.0.55.55/32的流量。

[R3] route-policy 1 permit node 10             
    [R3-route-policy] if-match ip-prefix 1
    [R3-route-policy] apply preferred-value 100

    [R3] route-policy 1 permit node 20

    [R3] bgp 100
    [R3-bgp] peer 10.0.2.2 route-policy 1 import

上述配置完成后，查看R3的BGP路由表。

[R3]dis bg routing-table 

    BGP Local router ID is 10.0.23.3 
    Status codes: * - valid, > - best, d - damped,
                h - history,  i - internal, s - suppressed, S - Stale
                Origin : i - IGP, e - EGP, ? - incomplete


    Total Number of Routes: 5
        Network            NextHop        MED        LocPrf    PrefVal Path/Ogn

    *>   10.0.33.33/32      0.0.0.0         0                     0      i
    *>i  10.0.44.44/32      10.0.1.1        0          100        0      200i
    * i                     10.0.2.2        0          100        0      200i
    *>i  10.0.55.55/32      10.0.2.2        0          100        100    200i
    * i                     10.0.1.1        0          100        0      200i

这里看到去往10.0.55.55的下一跳改成了10.0.2.2。

我们使用tracert看下报文转发的路径

这里我们可以看到,R1和R2 的路由在修改了R3上的Preferred Value值之后没有发生变化，说明了Preferred Value值只是作为本地路由器用来选择最佳BGP路由之用，不会传递给任何BGP邻居。
